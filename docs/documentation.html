Startup Landing Documentation

Overview of the project structure, dependencies, and customization tips.

Getting started

  1. Open the `pages/` directory to access ready-to-use HTML files.
  2. Update branding, copy, and images in the relevant sections.
  3. Serve locally with any static server (e.g., `npx serve pages`).

Structure

assets/
  css/      Core styles, responsive adjustments, and dark mode overrides
  js/       Vanilla JS utilities, optional plugins, and form handler
  img/      Organized folders for hero, features, team, and more
pages/      Marketing and auth pages using partial includes
partials/   Shared header, footer, sidebar, preloader, and modals
docs/       Documentation, changelog, and credits

Customization

  • Edit colors, shadows, and spacing via CSS variables in `assets/css/style.css`.
  • Toggle dark mode by setting the `dark-mode` class on `` or using the header switch.
  • Use `data-animate` attributes (`fade-up`, `fade-in`, `scale-in`) to animate sections when in view.
  • Add new partials in `partials/` and include them with `data-include="../partials/your-file.html"`.

JavaScript utilities

`assets/js/main.js` loads partials, handles navigation, dark mode, and scroll animations. `home2.js` adds counters and tabs for the SaaS variation. `form-handler.js` is a placeholder for integrating contact forms with third-party services (replace `FORM_ENDPOINT` with your API).

Assets

Replace placeholder images in `assets/img/` with your own. Keep file names consistent to avoid updating markup paths.

Deployment

Upload the project to any static hosting provider (Netlify, Vercel, GitHub Pages). Ensure relative paths remain intact.
